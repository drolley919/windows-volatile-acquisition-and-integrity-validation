# Findings

## Finding 1: Volatile Evidence Was Successfully Preserved

Critical volatile and semi-volatile system artifacts were successfully acquired prior to system shutdown. Collected evidence included a physical memory image, SAM registry hive, SYSTEM registry hive, and user-related artifacts.

The acquisition process ensured that transient system data was preserved for future forensic examination and investigative review.

## Finding 2: Memory Acquisition Completed Without Data Loss

A complete physical memory image was successfully captured from the live Windows system. The acquired memory image was stored within the designated forensic evidence repository and remained available for subsequent analysis activities.

No acquisition errors or collection failures were identified during the capture process.

## Finding 3: Registry Artifacts Were Successfully Collected

The SAM and SYSTEM registry hives were successfully preserved during live-response acquisition. These artifacts provide valuable information related to user accounts, authentication mechanisms, security settings, and system configuration.

Preservation of these files supports future forensic analysis and investigative correlation.

## Finding 4: Evidence Integrity Was Maintained

Cryptographic hash verification confirmed that acquired artifacts remained unchanged throughout the collection and storage process. Comparison of original and acquired hash values demonstrated no evidence of alteration, corruption, or data loss.

Integrity validation supports the authenticity and reliability of collected evidence.

## Finding 5: Evidence Documentation Was Complete

Hash inventories were successfully generated and reviewed for acquired artifacts. Documentation confirmed that collected evidence was properly cataloged and associated with corresponding cryptographic hash values.

Comprehensive documentation supports forensic reproducibility, chain-of-custody requirements, and future validation activities.
